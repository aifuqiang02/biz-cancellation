# 陕西省市场监管局网站自动化程序

基于 PyQt5 + QtWebEngine 实现的陕西省市场监管局网站 (snamr.shaanxi.gov.cn) 自动化管理系统，能够自动处理网站相关业务流程。

## 功能特性

- 图形化界面 + 内嵌浏览器
- 自动采集陕西省市场监管局网站办件信息
- 分页自动翻页，采集多页数据
- 智能解析办件列表，提取市场主体名称、各部门办理状态、申报时间、初审状态等完整信息
- 实时日志记录（包含JavaScript调试日志）
- 可配置的参数设置（轮次间隔、翻页等待、最大页数等）
- 全自动数据采集（检测→解析→翻页→输出JSON）
- JavaScript日志同步显示，无需打开控制台调试
- 完善的错误处理和调试信息
- 多重JavaScript回调兼容机制，支持不同PyQt5版本
- 完善的信号连接错误处理和降级机制
- 优化JavaScript注入时机，确保在页面完全加载后注入
- 移除复杂回调处理器依赖，使用稳定的备选日志方案
- 实现JavaScript日志缓冲区机制，定期同步日志到界面
- 修复JavaScript回调函数未定义错误
- 智能页面检测，即使JavaScript报告失败也会继续流程
- 精确的数据提取，只解析有效的办件信息

## 环境要求

- Python 3.10+
- Windows/Linux/macOS
- 已测试：PyQt5 + PyQtWebEngine（兼容性更好）

## 使用方法

1. **激活环境**：`activate_venv.bat`
2. **启动程序**：`python run.py`
3. **加载登录页面**：点击"加载登录页面"按钮访问陕西省市场监管局登录页面
4. **等待页面加载**：等待页面完全加载完成，JavaScript助手脚本注入完毕
5. **手动登录**：在浏览器中完成登录操作
6. **导航到办件页面**：手动导航到包含办件信息的页面
7. **配置参数**：设置轮次间隔、翻页等待、最大页数等参数
8. **调试页面**（可选）：点击"调试页面"按钮查看页面元素信息
9. **开始自动化**：点击"开始自动化"按钮（立即开始第一轮采集，完成后按设置的间隔自动执行后续轮次）
10. **监控日志**：查看实时操作日志和JSON数据输出

### ⚠️ 重要提醒

**必须先完成步骤3（加载登录页面）后再进行步骤9（开始自动化）**，否则程序会提示需要先加载页面。

## 工作原理

程序通过内嵌浏览器访问陕西省市场监管局网站，提供两种工作模式：

**手动模式**：
- 手动点击按钮进行各项操作
- 适合测试和单次操作

**自动化模式**：
- 点击"开始自动化"后立即开始第一轮采集，完成后按设置间隔执行后续轮次：
  1. 刷新页面
  2. 检查页面元素状态
  3. 执行配置的自动化操作
  4. 记录操作结果

### 📊 数据输出格式

每轮采集完成后，会在日志中输出如下格式的JSON数据：

```json
[
  {
    "name": "西咸新区秦汉新城聚百财百货店（个体工商户）",
    "status": {
      "市场监管": "正在办理",
      "税务": "未开始",
      "商务": "未开始",
      "海关": "未开始",
      "人社": "未开始"
    },
    "time": "2026-01-10",
    "review_status": "已初审",
    "page": 1,
    "collected_at": "2026-01-11 14:30:00"
  }
]
```

**字段说明：**
- `name`: 市场主体名称
- `status`: 各部门办理状态对象
- `time`: 申报时间
- `review_status`: 初审状态（如"已初审"、"待初审"等）
- `page`: 采集的页码
- `collected_at`: 数据采集时间戳

## 配置说明

- **轮次间隔**：每轮采集完成后等待时间，建议20-30分钟
- **翻页等待**：翻页后等待页面加载的时间，建议3-8秒
- **最大页数**：单轮采集的最大页数限制，建议50页以内
- **操作确认**：可开启/关闭敏感操作确认

### 故障排除

#### 虚拟环境问题
- **激活失败 ("系统找不到指定的路径")**：
  - 确保虚拟环境已创建：`python -m venv venv`
  - 检查路径是否正确，虚拟环境应在项目根目录下
  - 使用绝对路径：`call D:\path\to\project\venv\Scripts\activate.bat`

- **中文字符乱码**：
  - 脚本已修复编码问题，如仍有问题请使用英文输出版本
  - 或手动在命令行中设置：`chcp 65001`

#### 依赖安装问题
- **PyQt5安装失败**：
  - 使用预编译版本：`pip install PyQt5 PyQtWebEngine --only-binary=all`
  - 或升级pip：`python -m pip install --upgrade pip`

- **其他依赖失败**：
  - 重新运行：`pip install -r requirements.txt`
  - 检查Python版本 (需要3.10+)

- **程序已切换到PyQt5**：
  - PyQt5比PyQt6更稳定，兼容性更好
  - 如果遇到问题，请使用：`test_pyqt.py` 测试组件

#### 虚拟环境问题（重要）
- **显示(base)环境仍激活**：
  - 使用一键启动脚本：`start_program.cmd`（推荐）
  - 或重新打开CMD窗口运行：`activate_venv.bat`
  - 或使用：`python run_venv.py`（强制使用虚拟环境）

- **PyQt5-WebEngine安装失败**：
  - 运行：`start_program.cmd`（会自动检查并安装）
  - 或手动：`pip install PyQtWebEngine --force-reinstall`
  - 使用预编译版本：`pip install PyQtWebEngine --only-binary=all`

- **PowerShell环境混乱**：
  - 使用CMD而不是PowerShell：`start_program.cmd`
  - 或重新打开PowerShell窗口

#### 程序运行问题
- **无法加载页面**：检查网络连接和网站服务状态
- **脚本执行失败**：可能是页面结构变更，需更新选择器
- **登录失效**：重新手动登录网站

- **数据解析问题**：
- **找不到办件信息**：确认页面已正确加载并导航到办件列表页面
- **解析数据为空**：检查页面结构是否变更，可能需要更新解析逻辑
- **翻页失败**：确认翻页链接存在且可点击
- **调试解析**：点击"调试页面"按钮查看解析结果

- **JavaScript回调问题**：
- **`window.pyqtCallback is not a function`**：这是已修复的问题
- **`javaScriptWindowObjectCleared` 属性不存在**：PyQt5兼容性问题已修复
- **程序版本**：使用PyQt5版本，无需担心此错误
- **手动测试**：在浏览器控制台运行 `window.debugSnamrPage()` 测试函数
