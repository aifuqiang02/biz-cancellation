# 注销业务管理系统需求整理

## 项目背景
注销业务管理系统，主要用于管理企业注销流程，包括企业注销登记、状态跟踪、费用管理等。支持完整的注销业务流程管理。

## 核心需求

### 1. 注销业务管理
#### 1.1 注销业务列表
- [x] 注销业务创建、编辑、删除
- [x] 注销业务状态管理（待录入、已录入、待签字、初审驳回、已初审、已驳回、已完成、已终止）
- [x] 注销业务基本信息（企业名称、设立编号、费用金额、是否已付款）
- [x] 所有注销业务、任务，在一个页面显示。显示待执行任务总个数。可以展开任务。默认只显示5个任务。

#### 1.2 任务管理
- [x] 任务创建、编辑、删除（文本描述）
- [x] 任务状态管理（待处理/进行中/已完成）
- [x] 任务优先级设置（拖拽设置优先级、置顶）
- [x] 任务与注销业务关联（每个任务属于特定注销业务）
- [x] 任务列表展示（按注销业务分组，按优先级排序）

#### 1.3 业务关联
- [x] 每个注销业务独立管理任务状态
- [x] 注销业务间任务状态隔离
- [x] 注销业务切换时显示对应任务列表

### 2. 注销流程状态跟踪
#### 2.1 业务状态跟踪
- [x] 实时跟踪注销业务状态变化
- [x] 状态进度可视化（状态标签/进度指示器）
- [x] 业务结果状态显示（待录入/已录入/待签字/初审驳回/已初审/已驳回/已完成/已终止）
- [x] 创建时间和更新时间显示
- [x] 支持手动状态更新

#### 2.2 业务流程管理
- [x] 完整的注销业务流程管理
- [x] 状态流转控制和验证
- [x] 业务数据录入和修改
- [x] 费用管理和付款状态跟踪

#### 2.3 业务关联状态
- [x] 每个注销业务的流程状态独立显示
- [x] 当前业务的流程状态突出显示
- [x] 业务切换时自动切换状态视图
- [x] 多业务同时处理时的状态管理

#### 2.3 通知机制
- [ ] 浏览器通知API（需要用户授权）
- [ ] 页面内Toast通知（状态变更时自动提示）
- [ ] 声音提示（可选）
- [ ] 实时状态更新（状态变化推送）
- [ ] 通知内容包含业务名称和状态变更信息

#### 2.4 历史记录
- [ ] 每个注销业务的流程历史
- [ ] 业务完成率统计
- [ ] 状态变更记录
- [ ] 历史记录查看和管理功能

## 技术实现方案

### 架构设计
- **前端应用**: Vue.js 3 + TypeScript 单页面应用
- **后端服务**: Node.js + Express API服务
- **数据库**: PostgreSQL
- **通信方式**: WebSocket（实时状态更新）
- **部署方式**: 传统部署方式

### 技术栈详情
- **前端框架**: Vue.js 3 + TypeScript + Vite
- **状态管理**: Pinia
- **UI组件**: Element Plus
- **HTTP客户端**: Axios
- **路由**: Vue Router 4
- **后端框架**: Node.js + Express + Socket.io
- **数据库ORM**: Prisma
- **身份验证**: JWT + Passport.js

### 核心功能模块
1. **用户认证模块**: JWT身份验证
2. **注销业务管理模块**: 注销业务CRUD操作
3. **流程跟踪模块**: 业务状态跟踪和通知
4. **数据存储模块**: 数据库操作
5. **实时通信模块**: 状态变更推送


### 核心交互
1. 用户登录后进入注销业务管理主页
2. 可以创建新的注销业务，填写企业名称、设立编号、费用金额等信息
3. 每个注销业务可以添加和管理相关任务
4. 业务状态会根据实际处理情况进行更新
5. 支持业务状态的实时跟踪和通知
